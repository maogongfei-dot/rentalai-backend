# RentalAI — 项目状态（v1）

## 已完成的模块

- **Phase 5 第二轮 · 最小用户账户体验（阶段完成）**：顶栏 **Login / Sign Up**、已登录 **邮箱 + Logout**、**`/account`** 只读账户页；`RentalAIUserStore` 统一会话；分析历史按 **`guest` / `userId` 分桶**；访客提示（`guest_auth_hint`、`history_access_context`）不强制登录。
- **本地假登录 / 邮箱登录**：`/login`（本地演示 `current_user` 或邮箱 Bearer），受保护页统一顶栏导航。
- **AI 一句话分析**：`POST /api/ai-analyze` → 结果页展示结构化需求、推荐列表。
- **Explain/风险/决策**：解释文案、优缺点、风险提示、租/慎/不建议。
- **收藏与对比**：按 `user_id` 隔离的 `fav_list_*`，`/compare` 对比收藏与当前会话推荐。
- **分析历史**：结果页保存快照至 `analysis_history`，列表与详情页按用户过滤。
- **Demo 收口**：首页使用说明、结果页操作区、清空本地测试数据（开发用）。

## 当前阶段

**Demo / MVP 可用**：适合本地演示、验收与交付说明，**不是**生产级多租户 SaaS。

## 尚未做的内容（已知）

- **生产级账号**：邮箱验证、密码策略、HttpOnly 会话、刷新令牌、按用户的服务端权限模型（当前 API 多为**匿名可调用**，与 Phase 5 最小 auth **解耦**）。
- **服务端用户主数据与云端历史**：收藏/分析历史仍以 **本机 `localStorage` + 分桶** 为主；无多设备同步。
- 正式部署上的安全加固、审计与监控（除已有文档与可选 Webhook）。
- 多平台房源数据与抓取策略的规模化与合规流程。

## 下一阶段建议

1. **择一主线**：**后端正式用户持久化 + 受保护 API**（JWT/Cookie、用户级配额与审计）→ 或 **先继续产品功能**（历史管理、分析体验），auth 仍维持最小方案。
2. 若需跨设备：**云端历史同步**（与分桶模型对齐的迁移/合并策略）。
3. 为 `/api/ai/query` 等与异步任务增加限流与可观测性。
4. 按部署目标（内网 / 公网）补齐 HTTPS、Cookie 策略与 CSP。
